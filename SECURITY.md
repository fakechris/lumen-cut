# Security policy

## Reporting a vulnerability

Please report suspected vulnerabilities privately to the project maintainers.
Include the affected version, reproduction steps, impact, and any suggested
mitigation. Do not include real API keys, recordings, transcripts, or other
sensitive user data.

Please allow the maintainers time to validate and address the report before
public disclosure.

## Supported versions

Until the first stable release, security fixes are provided for the latest
published version only.
